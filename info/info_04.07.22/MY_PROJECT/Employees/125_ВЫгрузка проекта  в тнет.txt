1.Перед загрузкой проекта на сервер нужно проделать несколько манипуляций:

npm run build

по этой команде будет создаваться правильная, оптимизированная и сжатая сборка для загрузки на сервер.
Должна появится папка build с файлами:
-index.html - сформируется полностью оптимизированная структура
- Favicon (сокр. от англ. FAVorite ICON — «значок для избранного») — значок веб-сайта или веб-страницы.
 Отображается браузером во вкладке перед названием страницы, и в качестве картинки рядом с закладкой,
  а также в адресной строке в некоторых браузерах.
- manifest
- папка static

Проще говоря, все компоненты структура, скрипты - сжались и оптимизировались.

2. Вот эта папка build и есть та папка которую нужно загрузить на сервер.
Но перед загрузкой можно протестить как будет работать оптимизированный проект:

это можно сделать к примеру на mamp- сервере или open-server если нужно отправлять POST-запросы или установить nmp_ пакет:

npm install -g serve

или http-server инсталировать ( это тоже npm)

1. Логинимся
2. New app - создаю новый проект
3. Уже в проектевкладка Deploy 
- тут либо подключть репозиторий github или использовать менеджер от хероку Install the Heroku CLI скачивается по сслыке рядом.
Скачиваю и инсталирую
4. Далее в терменале проекта запускаю команду:

 heroku login
5. В проекте должен быть инициализирован git, если нет то нужно сделать и подвязать к гит-репазиторию и обязательно должен быть commit
6. Теперь нужно пядвязать гит-репозиторий с херокку, командой:

heroku git:remote -a test2203    ---- она прописана во вкладке deploy 

7. далее команды:
git add .
git commit -am "make it better"
git push heroku master

8. Далее на странице проекта в heroku жмем open app и проект должен запустится на сервере.
