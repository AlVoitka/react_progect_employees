

webpack.config.js 

Файл webpack.config.js - это файл в котором прописона как должен работать сборщик модулей webpack, этот файл нужно размещать в каждом проекте 
и подстраивать при необходимости . В нем я расписал что и как, но нужно следить за этим:



module.exports = {
  1) 
  mode: 'development',  ---- режим работы вэбрэка 'development' или 'production'

  ('production' - работает с оптимизацией кода и включает в себя различные инструменнты, режим для конечной сборки продукта
  'development' - более медленный и простой режим.
  Нужно при разработке ставить 'development', а при финишных рабтах 'production' - чтобы прогнать весь код и оптимизировать его)

  2)
  entry: './src/js/script.js',  ----- файл входа. путь к файлу-сборщику в который собираюся все части проекта

  3)
  output: {       ---файл выхода.
    filename: 'bundle.js',    ----так назавется вновьсозданный файл
    path: __dirname + '/dist/js/'   ---- место куда создастся файл bundle.js
  },
  я сказал что весь проект соберется будет в файле bundle.js
  тоесть при запуске в терменале  --- npx webpack сформируется  файл bundle.js в папке dist ->в папке js

  4)
  watch: true,  -- в этой позиции вэбпек отслеживает файлы и в авоматическом режиме собирает после каждого ctrl+s

  5)
  devtool: "source-map",  ---инструмен предназначенный исключительно для разработки.  навигация в браузере по подключенным модулям. 
  Те браузер подсасывае сжатый файл  из папки dist и в браузере во вкладке Sources ничего не видно по подключенным 
  к нему файлам ( modal/slider/), так вот с техгологией
  source-map я в консоли увижу все модуля и смогу по ним ориентироваться и отлаживать код в браузере.





И уже в конце следим за путем к этому файлу в inndex.html



    <script src="js/wow.min.js"></script>
    <script src="/dist/js/bundle.js"></script>   ----- вот берем подготовленный сжатый и собранный файл
