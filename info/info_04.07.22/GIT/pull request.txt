Pull request - по простому отправка своего кода в чужей репоз

к примеру
есть общий командный проект и я как разраб должен всенсти какие то провки в него

1.
Заходижу на чужей репоз в правом верхнем углу есть кнопка fork, нажимаю
и на моем аккаунте github создается клон этого проекта.

2.
Нужно склонировать его себе на комп
-создаю проект ( папку)
-запускаю vscode в созданном проекте и 

git clone https://github.com/yankovalenko94/Webdev2019/tree/master/Portfolio_step_2  

--- адрес копирую из поля пути браузера ( там где ввожу адрес сайта или поиска)


3.
нужно создать свою ветку в которой я буду вносить изменения

git checkout -b Alex-V  --- создал ветку Alex-V и переместился на нее 

git branch xxx ( создаю новую ветку с названием xxx, но остаюсь  на мастере)
git checkout xxx ( перехожу на ветку ххх)
git checkout –b yyy ( так я и создал ветку yyy и перешел на нее)
git branch -a (показывает все существующие в проекте ветки и реальную позицию ) 
git branch  тоже показывает все существующие в проекте ветки и реальную позицию
git branch –v  (Чтобы посмотреть последний коммит на каждой из веток)
Опции --merged и --no-merged могут отфильтровать этот список для вывода только тех веток, которые слиты или ещё не слиты в текущую ветку. 
Чтобы посмотреть те ветки, которые вы уже слили с текущей, можете выполнить команду git branch --merged:
    $ git branch --merged
     iss53
    * master

4.
После внесения изменеий нужно все закомитить и запушить в свой репоз

git add .
git commit -m ''
git push -u origin Alex-V  --- пушу именно на сваю ветку

5. Когда зайду на свой репоз должна появится кнопка Compare & Pool request  -- жму

там информация с какого репози ветки на кокой произойдет pull изменений в проекте

6. 
Хозяин получт письмо и сообщение что на его репоз сделан pool request
он может принять изменения или нет, ответить или прокомментировать мои изменения.
