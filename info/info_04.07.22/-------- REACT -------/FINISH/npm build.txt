------------------------------------   Vercel ---------------------------------------------------

Версель
Vercel — это облачная платформа, которая позволяет разработчикам размещать веб-сайты и веб-службы Jamstack, 
которые развертываются мгновенно, автоматически масштабируются и не требуют контроля, и все это без настройки. 
Они обеспечивают глобальную пограничную сеть, шифрование SSL, сжатие активов, аннулирование кеша и многое другое.

Шаг 1. Развертывание вашего проекта React в Vercel
Чтобы развернуть проект React с интеграцией Vercel для Git, убедитесь, что он был отправлен в репозиторий Git.

Импортируйте проект в Vercel, используя процесс импорта. Во время импорта вы найдете все соответствующие параметры, 
предварительно настроенные для вас, с возможностью изменения по мере необходимости.

После того, как ваш проект будет импортирован, все последующие отправки в ветки будут генерировать предварительные развертывания, 
а все изменения, внесенные в производственную ветку (обычно «мастер» или «основной»), приведут к производственному развертыванию.

После развертывания вы получите URL-адрес для просмотра вашего приложения в реальном времени, 
например следующий: https://create-react-app-example.vercel.app/.

Шаг 2 (необязательно): Использование личного домена
Если вы хотите использовать личный домен с развертыванием Vercel, вы можете добавить или перенести домен в настройках домена 
своей учетной записи Vercel.

Чтобы добавить свой домен в свой проект, перейдите к своему проекту на панели инструментов Vercel. После того, как вы выбрали 
свой проект, нажмите на вкладку «Настройки», затем выберите пункт меню «Домены». На странице домена вашего проекта введите домен,
 который вы хотите добавить в свой проект.

После добавления домена вам будут предложены различные способы его настройки.

Это сайт- инструкция как и куда лучше заливать папку build, но самый популярный у разрабов сервис - это https://www.heroku.com/
1. Логинимся
2. New app - создаю новый проект
3. Уже в проектевкладка Deploy 
- тут либо подключть репозиторий github или использовать менеджер от хероку Install the Heroku CLI скачивается по сслыке рядом.
Скачиваю и инсталирую
4. Далее в терменале проекта запускаю команду:

 heroku login
5. В проекте должен быть инициализирован git, если нет то нужно сделать и подвязать к гит-репазиторию и обязательно должен быть commit
6. Теперь нужно пядвязать гит-репозиторий с херокку, командой:

heroku git:remote -a test2203    ---- она прописана во вкладке deploy 

7. далее команды:
git add .
git commit -am "make it better"
git push heroku master

8. Далее на странице проекта в heroku жмем open app и проект должен запустится на сервере.
