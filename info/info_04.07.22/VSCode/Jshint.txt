JShint -- показывает в коде неправильный синтаксиз, ошибки . Его файлик есть у меня в папке plugin
Настройка:
на момент подключения должна быть настроена среда node.js
при работе с React личше отключать, там уже есть встроенный essinthConfig. Тогда не будет конфликтов.

При работе с REACT лучше отключать jshint (прямо в VSCode в extensions) - тк он будет конфоиктовать с esLint -который устанавливается при сборке Creat React App

в терминале :

npm i  -g jshint



сам файл .jshintrc

{
	"camelcase" : true,
	"indent": 2,
	"undef": true,
	"quotmark": false,
	"maxlen": 120,  --- максимальное колличество символов в одной строке
	"trailing": true,
	"curly": true,
	"strict": false,
	"browser": true,
	"devel": true,
	"jquery": true,
    "esversion": 9,
    "node": true
}

https://jshint.com/docs/options/ - тут  подробная документация

Помещать этот файл можно как в сам проект - в этом случае будет распространятся настройка только на проект,
а также в папку с множеством проектов и тагда один файлик будет работать на все проекты.
